# Documentation

## Overview

This directory contains technical documentation for the project.

The goal of this documentation is to provide clear guidance on system architecture, development workflow, and operational procedures.

---

## Structure

- `architecture.md` — System design and components
- `commands.md` — Development and operational commands
- `TODO.md` — Development roadmap and task tracking

---

## Purpose

This documentation is intended to:

- Help developers quickly understand the system
- Provide consistent development guidelines
- Serve as a reference for architecture decisions
- Simplify onboarding and collaboration

---

## Development Workflow

The project follows a structured, step-by-step development process:

1. Review tasks in `TODO.md`
2. Implement features incrementally
3. Use small, meaningful commits
4. Test changes in Docker environment

---

## Best Practices

- Keep code clean and modular
- Use environment variables for configuration
- Avoid hardcoding values
- Document important decisions

---

## Notes

- All services are containerized using Docker
- Backend and frontend are separated
- API-first approach is used across the system
